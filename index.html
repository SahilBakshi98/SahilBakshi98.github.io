Sem Eval 2019: OffensEval

Sahil Bakshi, Sayan Ghosh, Aman Raj

Online social media has become one of the most important ways to communicate and exchange ideas. Unfortunately, the discourse is often crippled by abusive language that can have damaging effects on social media users. In this project, we will work on identifying and categorizing offensive language in social media.

Problem Statement

The project consists of three tasks:

  • Task 1: Identify whether the given post contains offensive language.
  • Task 2: Identify whether the offensive language is targeted or non-targeted.
  • Task 3: Identify the target of the offence- Individual, Group or Other.

Baseline Methodologies Tried

We used two baseline classifiers for our task, one was Logistic Regression with a tf-idf model and the second was an SVM classifier with RBF kernel and GloVe vectors.
Both models use the hierarchical nature of the task and use reduced datasets for every further classification task.
Logistic Regression achieved an F1 score of 0.76, 0.51 and 0.5 on Subtasks A, B and C respectively.
SVM achieved F1 scores 0.72, 0.88 and 0.66 on Subtasks A, B and C respectively.

Approach

Preprocessing -
The first task was to clean and process the tweets present in our dataset which would result in better results for our classifiers. 4 basic preprocessing techniques were employed -
1. Tokenization of the tweets
2. Stopword removal
3. Stemming
4. Lemmatization
Hashtags and mentions were marked to be used as features in the classifier models.

Our deep learning classifiers use the hierarchical nature of the task to use trimmed datasets for further tasks. Subtask A uses the entire dataset for training. Subtask B and C use the trimmed datasets which do not contain unoffensive tweets.
Upsampling has been used for these subtasks to normalize the weightages.

We have used two deep learning classifiers, CNN and LSTM along with learning embedding layer and pretrained GloVe vectors.
1. LSTM -
Every tweet is converted to a one hot vector. Max length of tweet is set and padding with 0s is done for tweets less than the max length. The embedding matrix is then created and passed to the sequential layers. First we have the embedding layer with max length being 112. Then comes the LSTM layer with 100 nodes. Finally we have a dense layer with sigmoid activation function. Loss function used is binary crossentropy (for Subtask A) and sparse categorical crossentropy (for Subtask B and C)
2. CNN -
Preprocessing is done in the same way as with the LSTM. Embedding matrix is passed to the sequential layers. In the embedding layer, we have a word with 50 features in the output space and input length being equal to the max length of the tweets. Next we have a dropout layer for regularization with probability 0.2. Then we have a 1 dimension convolutional layer with 64 filters and kernel size being 5. The activation function used is relu. Next we have a 1 dimensional pooling layer with pool size 4. Finally we have a dense layer with sigmoid activation.
